# 市场调研Skill设计

### 需求分类逻辑

**核心目的：** 根据项目类型自动判断使用宏观还是微观调研方法，确保调研方向与业务目标匹配。

**适用场景：** AI助手在接到市场调研需求时，首先需要判断调研类型，再引导相应维度的数据采集。

#### 1. 分类决策树（判断维度）
* **业务类型判断：** 根据项目描述识别是全新业务还是现有产品迭代
* **判断标准：** 全新产品/新业务线→宏观；功能迭代/模块新增→微观
* **调研方法：** 通过问题清单引导PM确认：这是从0开始的业务，还是基于现有产品的功能优化？

#### 2. 宏观调研触发条件
* **核心判断：** 首次进入某个领域、需要验证商业模式可行性
* **具体表现：** 全新APP开发、全新业务线开辟、平台型产品从0到1构建
* **调研方法：** 自动询问："这是全新业务方向吗？如果是，启动宏观调研流程"

#### 3. 微观调研触发条件
* **核心判断：** 基于现有用户群体和产品生态的功能优化
* **具体表现：** 现有APP新增模块、功能优化体验提升、用户反馈需求实现
* **调研方法：** 自动询问："这是基于现有产品的功能迭代吗？如果是，启动微观调研流程"

#### 4. 自动分类机制
* **触发关键词：** 全新、首次、开辟、从零开始→宏观；新增、迭代、优化、模块→微观
* **分类流程：** 输入需求描述→关键词识别→确认分类→启动对应调研流程
* **调研方法：** NLP语义分析识别项目类型，提供明确的分类结果

### 交互流程总览

**核心目的：** 设计完整的用户交互路径，从需求输入到调研报告生成的全流程体验。

**适用场景：** PM使用AI助手进行市场调研时的完整操作流程设计和用户体验优化。

#### 1. 流程入口设计
* **触发方式：** 直接输入"帮我做市场调研"或上传需求文档
* **欢迎引导：** "您好！我是市场调研助手，请告诉我您需要调研的项目类型"
* **调研方法：** 自然语言输入 + 文件上传双通道，确保PM能快速开始

#### 2. 分类确认环节
* **分类展示：** 根据关键词显示判断结果："已识别为[宏观/微观]调研类型"
* **确认机制：** "这个分类是否正确？如不正确请重新描述您的需求"
* **调研方法：** 分类结果展示 + 人工确认机制，确保分类准确性

#### 3. 维度引导流程
* **维度展示：** 根据分类展示对应的调研维度（宏观：行业环境、市场规模等；微观：内部需求、用户反馈等）
* **跳过机制：** "是否需要跳过某些维度的调研？可选择性回答问题"
* **调研方法：** 维度列表展示 + 选择性跳过机制，提高调研效率

#### 4. 模拟用户环节
* **用户画像生成：** 根据调研结果生成典型用户画像
* **场景模拟：** "这个功能如何解决目标用户的XX问题？"
* **调研方法：** AI生成用户画像 + 场景化问题验证，确保调研实用性

#### 5. 联网搜索触发
* **触发条件：** 当PM需要最新数据或特定信息时
* **搜索范围：** 行业报告、竞品动态、用户反馈等
* **调研方法：** "是否需要联网搜索最新市场数据？" + 搜索结果摘要整合

#### 6. 报告生成流程
* **内容整合：** 自动整合所有调研维度的发现和结论
* **格式输出：** 生成完整的Markdown格式调研报告
* **完善建议：** "报告是否需要补充某个特定方面的深度分析？"
* **调研方法：** 模板化报告生成 + 可选的深度分析补充

---

**总结对比：**
* 分类逻辑：关键词识别 + 人工确认，确保调研类型准确性
* 交互流程：线性引导 + 选择性跳过，平衡全面性和效率
* 联网机制：按需触发 + 结果整合，保证数据时效性和实用性
* 报告生成：自动整合 + 可选完善，确保输出质量和PM满意度
